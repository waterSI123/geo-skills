# Report Structure

The MVP report must include these sections in this order.

## 1. Executive Summary

Use 5-8 concise bullets. Include:

- Current market-proxy GEO visibility state.
- Client brand position relative to tracked competitors.
- Biggest opportunity topic.
- Biggest content gap.
- Biggest risk.
- Recommended next action.

## 2. Project Scope

Show what the report is based on:

- client brand
- website
- product/service direction
- target roles
- market
- language
- platform
- competitors
- topic count
- prompt count
- analyzable response count
- brand-mentioning response count
- market-proxy response count
- market-proxy brand-mentioning response count

## 3. Market-Proxy GEO Visibility Scorecard

Include the required primary metrics:

- Market Visibility Score
- Weighted Market Visibility Score
- Market Share Of Voice
- Market Average Position
- Qualified Recommendation Rate
- Sentiment Score

For each metric, show:

- client value
- client rank
- leading competitor
- interpretation

## 4. Competitor Comparison

Compare the client against tracked competitors across:

- market visibility
- weighted visibility
- market share of voice
- average position
- qualified recommendation rate
- likely advantage

Explain whether the gap is mainly because the client is absent, lower ranked, weakly recommended, or less positively described.

## 5. Topic-Level Diagnosis

Prioritize high and medium opportunity topics. For each topic, include:

- topic
- opportunity level
- market client visibility
- weighted market visibility
- client absent rate
- weak recommendation rate
- strong competitors
- diagnosis
- recommended action

## 6. Content Gap Findings

Summarize content gaps separately from risks. Include:

- gap type
- severity
- why it matters
- evidence response IDs
- recommended action
- suggested content asset

## 7. Risk Findings

Include risk type, severity, evidence, business impact, recommended fix, and review-required status.

For `wrong_category_needs_review` and `wrong_feature_needs_review`, write that manual review is required before treating the issue as confirmed.

## 8. Prioritized GEO Action Plan

Create a short action plan with priority, action, mapped topic, reason, expected GEO impact, recommended owner, and suggested next skill.
